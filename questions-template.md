# 💬 プロジェクト質問テンプレート

各フェーズで確認すべき質問事項のテンプレート集です。プロジェクトの性質に応じて適切な質問を選択・カスタマイズして使用します。

## 📋 Phase 1: 要件定義フェーズの質問

### 🎯 プロジェクト全体に関する質問

```markdown
📌 **要件定義に関する確認事項**

### 1. プロジェクトの背景と目的
- このシステムを開発する背景・きっかけは何ですか？
- 解決したい具体的な課題は何ですか？
- 成功の判断基準（KPI）は何ですか？

### 2. ユーザーとステークホルダー
- 主要なユーザーは誰ですか？（複数可）
- 各ユーザーの利用シーンを教えてください
- システムに関わる他のステークホルダーはいますか？

### 3. 機能要件の優先順位
- 必須機能（Must Have）は何ですか？
- あれば良い機能（Nice to Have）は何ですか？
- 将来的に追加予定の機能はありますか？

### 4. 制約事項
- 予算の制約はありますか？
- 納期・スケジュールの制約はありますか？
- 技術的な制約（使用言語、フレームワークなど）はありますか？
- 法的・規制上の制約はありますか？

### 5. データとセキュリティ
- 扱うデータの種類と機密レベルを教えてください
- データの保存期間に関する要件はありますか？
- アクセス制御の要件はありますか？
```

## 💡 質問の使い方

### 1. 選択的使用
すべての質問を使う必要はありません。プロジェクトの性質に応じて、関連する質問のみを選択してください。

### 2. カスタマイズ
テンプレートの質問を基に、プロジェクト固有の質問を追加したり、不要な質問を削除したりしてください。

### 3. 段階的深堀り
最初は概要レベルの質問から始め、回答に応じてより詳細な質問を追加していきます。

### 4. 記録と追跡
すべての質問と回答は `.claude/qa/` ディレクトリに記録し、後から参照できるようにします。

## 📝 質問管理のベストプラクティス

1. **明確で具体的な質問**
   - 曖昧な表現を避ける
   - Yes/No で答えられる質問と、詳細説明が必要な質問を区別

2. **優先度の明示**
   - 必須回答と任意回答を区別
   - クリティカルな質問を先に配置

3. **回答の検証**
   - 矛盾する回答がないか確認
   - 不明確な回答は追加質問で明確化

4. **ドキュメント化**
   - 質問と回答のペアを整理して保存
   - 決定事項として後続フェーズで参照

5. **フィードバックループ**
   - 回答を基にした成果物をユーザーに確認
   - 認識の相違があれば早期に修正
